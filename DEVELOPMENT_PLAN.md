# Development Plan

## 1. MVP: 多主题静态学习舱

目标：验证交互式技术学习体验是否成立，并验证课程数据可以脱离具体主题扩展。

已完成：

- Ray 课程讲解
- Topic Hub 多主题入口
- `data/ray.json`
- `data/vllm.json`
- `data/topics.json`
- 架构图展示
- 章节导航
- 互动练习
- 答案解析
- 错题本
- 本地进度保存
- 一键启动脚本
- 章节完成状态
- 当前主题进度重置
- 多选题支持
- 填空题、代码阅读题、Debug 题
- `course.schema.json`
- 课程数据校验脚本
- 主题来源展示

后续补强：

- 增加更多 Ray 题型：场景设计题、开放问答题
- 给每道题增加难度、知识点、题型标签
- 补充 vLLM 官方来源和题库
- 增加题目难度和评分 rubrics
- 设计主题生成审核界面

## 2. 内容数据层

目标：把课程内容变成可生成、可审查、可版本化的数据。

计划：

- 定义 `course.schema.json`
- 支持主题、章节、概念、示例、练习、答案解析
- 支持引用来源字段，例如官方文档链接、论文、代码示例
- 增加内容质量检查脚本
- 支持多主题切换

候选数据结构：

```json
{
  "topic": "Ray",
  "lessons": [],
  "questions": [],
  "sources": []
}
```

## 3. Survey Agent

目标：让 agent 自动调研技术主题，并输出结构化课程数据。

计划：

- 输入：技术主题，例如 Ray、vLLM、Kubernetes
- 资料源：官方文档、GitHub、论文、示例代码、可靠博客
- 输出：主题地图、核心概念、学习路径、练习题素材
- 记录来源，避免内容不可追溯
- 增加人工审核入口

关键原则：

- 优先官方资料
- 每个概念都要能追溯来源
- 练习题必须有明确答案和解释
- 对版本敏感的内容要标注版本

## 4. Course Builder Agent

目标：把 survey 结果转成适合学习的课程。

计划：

- 自动拆分章节
- 生成由浅入深的讲解
- 生成概念卡片
- 生成代码示例
- 生成练习题和答案解析
- 标注知识点依赖关系

课程形态：

- 概念讲解
- 代码阅读
- 场景判断
- 架构推演
- 错题复盘

## 5. Interactive Tutor

目标：让应用从“做题系统”进化成“会追问的老师”。

计划：

- 支持自由回答
- 根据回答判断掌握度
- 针对错误进行追问
- 支持提示而不是直接给答案
- 将学习者的回答写入学习记录
- 自动生成复习建议

## 6. 技术演进

当前：零依赖静态页面。

下一阶段：

- 迁移到 Vite + React
- 使用 TypeScript 管理课程数据和状态
- 增加路由：主题页、课程页、练习页、复盘页
- 使用本地 JSON 文件作为课程数据源
- 增加 Playwright 测试

再下一阶段：

- 增加后端 API
- 保存用户学习记录
- 支持 agent 任务队列
- 支持主题生成和人工审核
- 支持部署到 Vercel 或 Cloudflare Pages

## 7. 主题扩展

优先主题：

- Ray
- vLLM
- Kubernetes
- CUDA
- Transformer
- LangGraph
- OpenAI API

每个主题都应包含：

- 架构总览
- 核心概念
- 最小可运行示例
- 常见误区
- 工程实践
- 综合练习
